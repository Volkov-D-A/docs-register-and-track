# User Scenarios

Дата аудита: 2026-05-07

## Основные Сценарии

1. Первый запуск и первичная настройка:
   администратор запускает миграции, создает первого пользователя `admin`, задает организацию в настройках.

2. Вход и профиль:
   пользователь входит по логину/паролю, меняет пароль, обновляет профиль, выбирает тему.

3. Администрирование пользователей:
   администратор создает пользователей, назначает подразделение, включает участие в документообороте, выдает системные и документные права, сбрасывает пароли.

4. Настройка справочников:
   администратор управляет номенклатурой и подразделениями; пользователь с `references` управляет организациями и исполнителями резолюций.

5. Регистрация входящего письма:
   пользователь с правом `create` выбирает номенклатуру, дату, тип документа, корреспондентов, содержание, подписанта, резолюцию и сохраняет документ.

6. Регистрация исходящего письма:
   пользователь выбирает номенклатуру, получателя, адресата, дату, содержание, подписанта и исполнителя отправителя.

7. Регистрация обращения граждан:
   пользователь фиксирует дату регистрации, дату обращения, ФИО и адрес заявителя, вид обращения, категорию, листы, корреспондентов и резолюции.

8. Регистрация приказа:
   пользователь фиксирует номер/дату, заголовок, контроль исполнения, срок, активность/отмену и список ознакомления.

9. Просмотр и поиск документов:
   пользователь фильтрует реестры, открывает карточку документа, смотрит вложения, поручения, ознакомления, журнал и связи.

10. Редактирование документа:
    пользователь с правом `update` меняет данные документа; действие пишется в журнал.

11. Поручения:
    пользователь с правом `assign` создает/редактирует/отменяет поручения; исполнитель принимает в работу, завершает, получает возврат, прикладывает отчет.

12. Ознакомления:
    пользователь с правом `acknowledge` создает задачу ознакомления; адресаты отмечают просмотр и подтверждение.

13. Вложения:
    пользователь с правом `upload` загружает, скачивает, открывает и удаляет файлы; исполнитель может прикладывать файлы при завершении поручения, если это включено настройкой.

14. Связи документов:
    пользователь с правом `link` связывает документы, удаляет связи, просматривает граф связей.

15. Статистика:
    пользователь с соответствующими статистическими правами смотрит отчеты по документам, поручениям и системе.

16. Операционное обслуживание:
    администратор просматривает миграции, запускает/откатывает миграции; оператор выполняет backup/restore PostgreSQL и MinIO скриптами.
