# Terms Glossary

Дата аудита: 2026-05-07

## Предметная Область

- Документ: общая сущность в таблице `documents`, имеющая вид, номенклатуру, регистрационный номер, дату, тип, содержание, автора и timestamps.
- Вид документа: системный код документа: входящее письмо, исходящее письмо, обращение граждан, приказ.
- Тип документа: фиксированное значение вроде "Письмо", "Договор", "Акт", "Обращение", "Приказ".
- Номенклатура: дело/индекс для регистрации документов конкретного года и вида; хранит режим нумерации и следующий номер.
- Регистрационный номер: номер документа, уникальный в пределах вида и года регистрации.
- Корреспондентская регистрация: внешний номер, дата и организация-корреспондент для входящих писем и обращений.
- Резолюция: текст поручения/указания, автор и исполнители резолюции по документу.
- Организация: справочник корреспондентов и получателей.
- Исполнитель резолюции: справочник имен исполнителей резолюции.
- Подразделение: организационная единица пользователя, связанная с доступной номенклатурой.
- Поручение: задача по документу с исполнителем, соисполнителями, сроком, статусом, отчетом.
- Соисполнитель: дополнительный пользователь, участвующий в поручении.
- Ознакомление: задача для пользователей прочитать/подтвердить документ.
- Лист ознакомления приказа: список ФИО по приказу с отметкой ознакомления.
- Вложение: файл документа; metadata в PostgreSQL, содержимое в MinIO.
- Связь документов: отношение source -> target с типом `reply`, `follow_up`, `related`, `clarification`, `order_amends`, `order_cancels`.
- Журнал документа: история действий по документу.
- Административный аудит: журнал действий администрирования пользователей, справочников, настроек и миграций.

## Права и Профили

- System permission: глобальное право пользователя: `admin`, `references`, `stats_documents`, `stats_assignments`, `stats_system`.
- Document permission: право действия по виду документа: `create`, `read`, `update`, `assign`, `acknowledge`, `upload`, `link`, `view_journal`.
- Участник документооборота: пользователь с `is_document_participant=true`; получает ограниченный доступ через подразделение, поручения и ознакомления.
- Администратор: пользователь с `admin`.
- Делопроизводитель/регистратор: пользователь с document-domain правами на создание/чтение/изменение документов.
- Исполнитель: пользователь, работающий с поручениями и ознакомлениями.
- Смешанный профиль: пользователь с правами регистратора и участием в документообороте.
- UX-профили `admin`, `clerk`, `executor`, `mixed`: исторические названия бывших бизнес-ролей; после рефакторинга используются только для выбора интерфейсного поведения, а не для принятия решений доступа.

## Технические Термины

- Wails Bind: Go service, экспортированный во frontend через Wails.
- DTO: frontend-facing структура из `internal/dto`.
- Embedded migrations: SQL migrations, встроенные в бинарник.
- Embedded frontend assets: собранный `frontend/dist`, встроенный в бинарник.
- Seq: внешний сервис приема структурированных логов.
- MinIO bucket: объектное хранилище вложений.
- Production candidate: текущее чистое состояние проекта, прошедшее базовую сборку и тесты в окружении аудита.
