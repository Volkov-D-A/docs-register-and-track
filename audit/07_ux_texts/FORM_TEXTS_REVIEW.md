# Form Texts Review

Дата аудита: 2026-05-28
Этап: H.02, H.05

## Основные Замечания

### Вид документа / тип документа

Текущий UI использует `Вид документа` для `documentTypeId` в document forms, хотя glossary различает:

- `Вид документа`: входящее письмо, исходящее письмо, обращение граждан, приказ.
- `Тип документа`: Письмо, Договор, Акт, Обращение, Приказ.

Предложение: в формах `documentTypeId` использовать label `Тип документа`; модалки выбора incoming/outgoing/appeal/order оставить `Выберите вид документа`.

Риск изменения смысла: низкий, но нужно согласовать с бизнес-терминологией.

### Дело / номенклатура

Формы используют `Дело`, карточка документа показывает `Номенклатура`, фильтр говорит `Все дела`.

Предложение:

- user-facing field: `Дело`;
- admin/reference context: `Номенклатура дел`;
- first-use help: `Дело из номенклатуры, по которому ведется регистрация`.

Риск: средний, потому что "номенклатура" может быть официальным термином.

### Сокращения

Examples:

- `ПОС`;
- `Рег. №`;
- `Проср.`;
- `< 3 дней`.

Предложение:

- `Поступило через ПОС` with tooltip/help `Платформа обратной связи`;
- `Регистрационный №`;
- `Просроченные`;
- `Ближайшие 3 дня`.

### Placeholders

Examples:

- `Поиск...`;
- `ФИО`;
- `Введите номер документа`;
- `Что нужно сделать...`.

Предложение: use specific placeholder only when it adds an example:

- `Поиск по номеру, содержанию или исполнителю`;
- `Например, Иванов И.И.`;
- `Например, 01-12/45`;
- `Опишите, что нужно выполнить`.

## Safe Text Replacements

| Current | Proposed |
| --- | --- |
| `Кол-во листов` | `Количество листов` |
| `Рег. №` | `Регистрационный №` |
| `Проср.` | `Просроченные` |
| `< 3 дней` | `Ближайшие 3 дня` |
| `Не действующий` | `Недействующий` |
| `Контролер` | `Контролёр` or `Контролирующий` |
