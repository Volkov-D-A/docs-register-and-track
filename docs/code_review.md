# Code Review

## Status

Основные document-domain проблемы из первого ревью уже закрыты:

- `LinkService` переведен на `clerk`-only
- документы проверяют доступ на backend, а не только на уровне списков
- вложения больше не скачиваются в обход document-domain доступа
- поручения и ознакомления больше не полагаются на фронтенд как на механизм безопасности
- `admin` выведен из document-domain в backend и frontend
- admin-domain сервисы переведены на `activeRole`

## Findings

Явных актуальных находок по ранее поднятым пунктам сейчас не осталось: document-domain и admin-domain сервисы переведены на модель `activeRole`, а соответствующие тесты добавлены.

## Gaps

Полезно продолжить ревизию уже не по ролям, а по бизнес-границам: например, отдельно проверить batch-операции, экспорт/импорт и системные задачи на предмет того, не требуют ли они более узких policy-helper'ов вместо прямой проверки роли.

## Verification

Проверки после изменений:

- `GOCACHE=/tmp/go-cache go test ./...`
- `npm run build`
