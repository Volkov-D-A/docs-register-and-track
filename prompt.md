# Система регистрации входящих, исходящих документов, а также контроля сроков исполнения поручений по этим документам
## Назначение системы

1. Регистрация входящих документов
2. Регистрация исходящих документов
3. Назначение и контроль сроков исполнения поручений

## Ролевая модель

1. администратор - управляет справочниками, пользователями, правами доступа
2. делопроизводитель - регистрирует входящие и исходящие документы, направляет документы на исполнение, осуществляет контроль исполнения поручений
3. исполнитель - исполняет поручения, следит за сроком исполнения назначенных ему поручений

## Ключевые особенности системы

1. Автоматическая нумерация документов по номенклатуре дел (формат: `ИНДЕКС/№`)
2. Дашборд с ролевой адаптацией
3. Прикрепление файлов (PDF, DOC, DOCX, XLS, XLSX, JPG, PNG, TXT)
4. Связь исходящих/входящих документов для получения информации о прохождении документов
5. Справочники отправителей и получателей заполняются автоматически уникальными записями при регистрации документов делопроизводителем
6. Возможность пользователю иметь несколько ролей (например администратор может выступать в роли исполнителя) и переключаться между ними

## Основные экраны

1. Дашборд - в зависимости от роли пользователя имеет разные представления
2. Журнал входящих документов (недоступен для исполнителя)
3. Журнал исходящих документов (недоступен для исполнителя)
4. Журнал поручений (доступен делопроизводителю, исполнителю в части своих поручений)
5. Параметры и управление справочниками (доступен только администратору)
6. Профиль пользователя

## Номенклатура дел

1. Справочник номенклатуры дел ведется в системе администратором
2. Дела имеют следующую структуру:

* наименование дела
* индекс дела
* год
* направление (входящий, исходящий)

3. Номенклатура дел формируется из созданных дел и действует в календарном году указанном при создании
4. Нумерация документов ведется независимо по каждому делу номенклатуры

## Входящие документы

### Структура входящего документа

* исходящий номер отправителя, дата отправки
* входящий номер получателя, дата получения
* входящий номер промежуточный, входящая дата промежуточная (опционально)
* тип документа (письмо, договор, акт, счет, etc)
* тема документа
* количество листов
* содержание документа
* организация отправитель
* подписант отправителя
* исполнитель отправителя
* организация получатель
* адресат (кому направлен)
* резолюция

### Регистрация входящего документа

1. Входящий документ регистрируется в системе делопроизводителем и ему присваивается входящий номер получателя
2. Входящий номер присваивается автоматически в момент регистрации документа на основании номенклатуры дел и порядкового номера в этой номенклатуре дел. Номер присваивается в формате ИНДЕКС/Порядковый номер (например 01-05/123)
3. Поля обязательные для регистрации документа:

* исходящий номер отправителя, дата отправки
* тип документа (письмо, договор, акт, счет, etc)
* тема документа
* количество листов
* содержание документа
* организация отправитель
* подписант отправителя
* исполнитель отправителя
* организация получатель
* адресат (кому направлен)

## Исходящие документы

### Структура исходящего документа

* исходящий номер отправителя, дата отправки
* входящие номер и дата в ответ на который подготовлен исходящий документ (опционально)
* тип документа (письмо, договор, акт, счет, etc)
* тема документа
* количество листов
* содержание документа
* организация отправитель
* подписант отправителя
* исполнитель отправителя
* организация получатель
* адресат (кому направлен)

### Регистрация исходящего документа

1. Исходящий документ регистрируется в системе делопроизводителем и ему присваивается исходящий номер.
2. Исходящий номер присваивается автоматически в момент регистрации документа на основании номенклатуры дел и порядкового номера в этой номенклатуре дел. Номер присваивается в формате ИНДЕКС/Порядковый номер (например 01-05/123).
3. В случае если исходящий документ регистрируется в ответ на входящий документ, то в исходящем документе указывается входящий номер и дата входящего документа, также в системе фиксируется связь между исходящим и входящим документом.
4. Поля, обязательные для регистрации исходящего документа:

* исходящий номер отправителя, дата отправки
* входящие номер и дата в ответ на который подготовлен исходящий документ (опционально)
* тип документа (письмо, договор, акт, счет, etc)
* тема документа
* количество листов
* содержание документа
* организация отправитель
* подписант отправителя
* исполнитель отправителя
* организация получатель
* адресат (кому направлен)

## Поручения
### Создание поручения

1. Делопроизводитель на основании резолюции руководителя создает поручение
2. У поручения есть ответственный исполнитель, также может быть соисполнитель (один или несколько)
3. У поручения может быть срок исполнения
4. Для одного документа может быть несколько поручений с разными исполнителями и/или сроками исполнения

### Контроль поручений

1. Контроль применяется к поручениям, если указан срок исполнения поручения.
2. У делопроизводителя есть фильтры в журнале входящих документов по срокам исполнения.
3. У исполнителя информация о поручениях и сроках их исполнения находится на дашборде и в журнале поручений
